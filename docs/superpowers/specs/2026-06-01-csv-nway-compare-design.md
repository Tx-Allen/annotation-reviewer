# CSV N-way 横向对比 — 设计

**日期:** 2026-06-01
**目标文件:** `compare.html`(原地升级,自包含单文件,纯本地不上传)
**状态:** 已批准设计,待写实现计划

## 背景与现状

`compare.html` 现为严格 **2-way** 对比:两个硬编码文件输入 `① 旧 CSV(基准)` / `② 新 CSV(对比对象)`,状态 `S = { old, neu, keyCol, diff }`,`compare()` 按键列做成对 diff(`changed`/`same`/`removed`,新键 `added`),表格列为 `键列 | 状态 | 各公共字段`,单元格显示 `旧 → 新`。

需求:支持**多于 2 份** CSV 的横向对比。

## 确定的决策

1. **对比模型 = 共识并排**:不设基准,按键列对齐后把 N 份的同一字段值并排,谁不一致就高亮。
2. **布局 = 按字段分组**:一行 = 一条记录(键);每个字段下再拆 N 个子列(每份一列),同字段的多份值紧挨。
3. **共识/多数票 = 核心版**:每个 `键×字段` 算多数值 + 吻合比例(如 `2:1`),与多数不符的单元格单独高亮(离群);行状态按是否全体一致分 `一致/分歧`。**不含**离群率统计、不含"多数值"单独融合导出(这是被排除的增强项)。
4. **N=2 统一为特例**:不保留旧的"旧→新方向 diff"视图;2 份也走并排+高亮(1:1 视为无多数)。一套代码路径。

## 数据模型

- `S.files`:数组,每项 `{ id, label, name, header[], rows[] }`,取代 `S.old/S.neu`。
  - `label` 自动分配 `文件1/文件2/…`(短标签,用于 N 个窄子列的表头);完整文件名显示在图例/状态行。
- `S.keyCol`、`S.diff` 保留。
- 最少 2 份。UI:`＋ 添加 CSV` 动态加输入槽,每槽带 `×` 移除(降到 2 以下禁止移除)。软上限约 8——超出仍可用,仅提示"并排列会变窄";无硬上限。

## 键列与字段

- 键列下拉 = **所有已载入文件 header 的交集**;默认 `annotation_id`(若在交集内),否则交集首列。(把现有"两份交集"逻辑推广到 N 份。)
- 对比字段 = 键列交集 − 键列。
- 仅部分文件拥有的列 → 在 note 区列出"(对比忽略)",推广现有 `onlyOldCols/onlyNewCols` 提示。
- 交集为空 → 禁用键列下拉并提示"N 份没有公共列,无法对齐"。

## 对比算法(共识模型)

- 每份文件按键列建 Map(沿用 `buildMap`:重复键保留首次出现 + 计数,推广到 N 份)。
- 键全集 = N 份所有键的并集。对每个键 `k`:
  - 对每个对比字段 `f`:收集 `vals[fileId]`,该文件无键 `k` 时为 `∅`(缺失)。
  - **多数票**:仅对*存在的值*计数,最高频者为多数值,显示吻合比例(如 `2:1`、`3:0`);**与多数不符的单元格高亮(离群)**;`∅` 计为不一致。
  - **平票/无多数**(含 N=2 的 `1:1`、以及偶数平分):标"无多数",所有相异值都高亮,不指定多数值。
  - 字段一致 = 所有存在值相等且无缺失。
  - **行状态**:全部对比字段一致 → `一致`;否则 `分歧`。原 `added/removed` 被吸收——某键只在部分文件出现 → 缺失格显示 `∅` + 行内小标"缺 文件B",行记 `分歧`。
- 排序:`分歧` 行在前,`一致` 行在后。

## 表格布局(按字段分组)

- 两层表头:
  - 第一层:`键列 | 状态 | [字段1 跨 N(或 N+共识)列] | [字段2 …] | …`
  - 第二层:每个字段组下为各份的 `label`(可选末列 `共识`,显示多数值 + 比例)。
- 单元格:正常值;离群值高亮(沿用现有 `--chg/--del` 配色);`∅` 用灰 `--same`。
- 保留"只看分歧(隐藏全体一致行)"开关,默认开;保留"显示 X / Y 行"计数。
- 文件图例:`label → 完整文件名 · 行数 · 列数`,放结果卡顶部。

## 导出

- "导出分歧 CSV":只导出 `分歧` 行。
- 列 = `键列, __status,` 然后每个对比字段:每份一列 `f__<label>` + `f__majority`(多数值,无多数留空)+ `f__agree`(如 `2:1`)。
- 这里的 `f__majority`/`f__agree` 只是把屏幕上的共识列一并导出,**不是**下方非目标里说的"融合数据集"。
- 沿用 `csvQuote` 防 CSV 注入;沿用 `downloadFile`(带 BOM)。

## 复用(不改动)

`parseCSV` / `csvToObjects` / `sniffDelim` / `deformula` / `csvQuote` / `downloadFile` / 主题切换 / CSP 友好(无内联事件处理器)/ 纯本地不上传 —— 全部保留。改动集中在:顶部 markup 的输入区(静态两槽 → 动态槽)、`<script>` 中的状态对象与 `compare/render/exportDiff`。

## 测试

- 抽出内嵌 `<script>` 跑 `node --check`,防模板/转义折叠(参考 dashboard 模板字面量踩坑)。
- 造小样本 CSV 覆盖:全体一致行、`2:1` 多数、平票(`1:1`/`2:2`)、某键仅在部分文件、仅部分文件拥有的列、重复键、含 `=`/前导符的注入值、TSV 分隔符。
- 验证:行状态、离群高亮位置、多数值与比例、缺失 `∅` 与行内标注、"只看分歧"过滤、导出列结构。

## 非目标(YAGNI)

- 不做每份文件的离群率统计("谁最常偏离多数")。
- 不做单独的"融合数据集"导出(把整表用多数值解析成一份干净的一行一键结果 CSV)。分歧导出里的 `f__majority` 列不是这个。
- 不保留旧的 2 份方向性 diff 视图。
- 不做服务端/上传;保持纯客户端单文件。
